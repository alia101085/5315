MongoDB Document Database Queries with AWS EC2 Server

 

(1)   what is Document Database,

A document database is designed to store semi-structured data as documents, typically in JSON or XML format. Unlike traditional relational databases, the schema for each non-relational (NoSQL) document can vary, giving Developers, Database Administrators, and IT Professionals more flexibility in organizing and storing application data and reducing storage required for optional values.

 

(2)   what is MongoDB and identify the benefits of MongoDB,

MongoDB is NoSQL document database that provides high performance, high availability, and easy scalability. Some of the benefits are:  It can store heterogeneous data(JSON objects), Deep query-ability. MongoDB supports dynamic queries on documents using a document-based query language that's nearly as powerful as SQL, Ease of scale-out: MongoDB is easy to scale, No complex joins, Structure of a single object is clear.

 

(3)   why SQL queries and queries in MongoDB are different

Since MongoDB is document-oriented semi-structured database and SQL is table based structured database, the queries differ to handle the nested constructs involved in JSON storage elegantly. Due to the document-oriented nature of the storage, if it were to do SQL, it really world be a variant, not true SQL. The extensions would need something like the current MongoDB dot notation to reach into objects. Development is simplified as MongoDB documents map naturally to modern, object-oriented programming languages. Using MongoDB removes the complex object-relational mapping (ORM) layer that translates objects in code to relational tables. MongoDB’s flexible data model also means that your database schema can evolve with business requirements. For example, schema changes that took days of weeks in The Weather Channel's MySQL databases could be made in just hours with MongoDB.
