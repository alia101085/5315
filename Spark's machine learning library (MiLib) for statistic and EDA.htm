Spark's machine learning library (MiLib) for statistic and EDA

 

Background

This use case is for exploratory analysis goes beyond simple analytics. We use Spark's machine learning library MlLib for basic statistics functionality and to explore and better understand our dataset.

 

Dataset used

I use reduced or 10% of KDD Cup 1999 dataset which has 2.1M observation or 75M uncompressed data. KDD cup dataset was used for The Third International Knowledge Discovery and Data Mining Tools Competition, which was held in conjunction with KDD-99 the Fifth International Conference on Knowledge Discovery and Data Mining. The competition task was to build a network intrusion detector, a predictive model capable of distinguishing between ``bad'' connections, called intrusions or attacks, and ``good'' normal connections. This database contains a standard set of data which includes a wide variety of intrusions simulated in a military network environment.

Insights

The major insight from using Spark Machine learning and Data Wrangling tools in the KDD data frame showed us which variables are highly correlated. The strong correlated data can be used to create model selection and make predictions. We know from this use case that dst_host_same_src_port_rate references the percentage of the last 100 connections to the same port, for the same destination host. In the resulted correlation matrix shows it’s also positively correlated to number of bytes sent form source to destination and the number of connections to the same service as the current connection in the past 2 seconds.

def parse_interaction(line):

    line_split = line.split(",")

    return np.array([float(x) for x in clean_line_split])
